# Audio Visualizer Widget

Windows 音频桌面可视化组件原型。

## 项目结构

- `src/`: React + Canvas 前端界面。
- `src-tauri/`: Tauri + Rust 桌面后端。
- `src-tauri/src/audio/`: Windows WASAPI loopback 音频采集与分析。
- `scripts/stop-dev.ps1`: 停止本项目开发进程和 1420 端口占用。
- `.npmrc`: npm 使用 `https://registry.npmmirror.com/`。
- `.cargo/config.toml`: Cargo 使用中科大 crates.io 稀疏索引镜像。

### v0.2 系统音频采集第一版

已完成：

- Rust 后端接入 Windows WASAPI loopback。
- 捕获默认播放设备的全系统输出音频。
- 将真实 PCM 音频分析成 `AudioFrame`。
- 保持前端事件接口 `audio-frame` 不变。

当前限制：

- 这一版是全系统音频，不区分具体进程。
- 暂未做音频设备切换监听；切换耳机/扬声器后可能需要重启应用。

### v0.3 波形视觉优化和中文设置

### v0.5 顶部样式工具栏和 X 轴长度

已完成：

- 将样式设置从右上角浮窗改为顶部平铺工具栏，避免小窗口裁切。
- 增加 `X轴长度` 设置，可以拉长或拉短波形横向显示密度。
- 默认 X 轴长度调长，让波形显示更多起伏。
- 波形算法改为更柔和的中心强化，两端配合渐隐弱化。

### v0.6 FFT 频谱波形第一版

已完成：

- 后端加入 `rustfft`。
- WASAPI 采集到的 PCM 音频会先混合成单声道，再做 FFT 分析。
- FFT 使用 2048 点采样窗口和 Hann 窗，降低频谱跳动和边缘泄漏。
- `bins` 从时域能量升级为真实频率能量。
- 频谱输出扩展为 96 个点，并按对数频率分布覆盖约 32Hz 到 18kHz。
- 前端同步为 96 个频率点，左侧偏低频、右侧偏高频。
- 前端去掉旧的中间裁切采样，完整绘制低频到高频的频谱范围。
- 平滑参数调得更稳，降低之前那种大幅抖动。

当前说明：

- 现在已经是 FFT 频谱驱动，但视觉上仍然画成一条波浪线，而不是传统柱状频谱。
- `X轴长度` 目前控制的是视觉密度和波浪拉伸，不是严格的频率缩放。
- 默认还是监听系统默认输出设备，不区分具体应用进程。

### v0.7 频谱轮廓增强

已完成：

- 后端频谱分析从单纯平均能量改为 `平均值 + 峰值` 混合，减少频段被压平的问题。
- 后端增加动态对比处理，突出当前帧里真正有能量的频段。
- 前端增加频谱轮廓塑形，把低能量区域压回中线，把局部峰值抬起来。
- 减弱空间平滑，保留更多频率起伏细节。
- 波形两端增加安静权重，低频和高频区域默认更贴近中线，只有能量明显时才会跳动。
- 预览数据也改成更接近真实频谱峰值的形态，方便浏览器预览调样式。

当前说明：

- 这一版目标是更接近参考图里“多数区域平、少数频段明显起伏”的波浪条。
- 视觉效果会比 v0.6 更有高低差，但具体强度还需要用不同音乐实测微调。

### v0.8 波形清晰度优化

已完成：

- 降低光晕层的不透明度和模糊半径，避免颜色被光晕糊开。
- 主波形取消阴影模糊，改为清晰渐变实线。
- 主波形下方增加轻微暗色描边，提高复杂壁纸上的对比度。
- 主波形上方增加细高光线，让颜色边缘更利落。
- 透明度设置最低值从 `0.35` 提高到 `0.55`，避免波形长期过淡。

当前说明：

- 这一版优先保证线条清晰度，光晕只作为背景衬托。
- 如果本地之前保存过较低透明度，重新打开后会被限制到新的最低透明度。

### v0.9 显示模式、点击穿透和位置记忆

已完成：

- 样式设置新增 `显示` 选项，可在 `波浪条` 和 `柱状图` 之间切换。
- 柱状图使用圆角竖条和渐隐边缘，视觉接近桌面音乐频谱条。
- 样式设置新增 `允许拖动`，默认开启。
- 关闭 `允许拖动` 后，窗口在设置面板关闭时会启用点击穿透，鼠标可以点到下面的桌面文件或窗口。
- 从托盘再次打开样式设置时，会临时关闭点击穿透，避免无法重新操作设置。
- 记录窗口拖动后的位置，下次打开自动恢复到上次位置。
- 样式设置继续保存在本地存储，下次打开自动沿用。
- 顶部工具栏移除 `样式设置` 标题和文字关闭按钮，改为最右侧关闭图标，节省横向空间。

当前说明：

- 点击穿透依赖 Tauri 窗口权限 `setIgnoreCursorEvents`。
- 窗口位置通过前端本地存储保存；清理应用数据后会恢复默认位置。

## 运行前端预览

```powershell
npm run dev
```

打开：

```text
http://127.0.0.1:1420/
```

浏览器里会使用预览数据；Tauri 桌面环境里会优先监听 Rust 后端推送的 `audio-frame` 事件。

## 运行桌面应用

```powershell
npm run desktop
```

当前窗口是透明波形组件：

- 按住波形区域左键可拖动位置。
- 右键系统托盘图标可打开中文菜单。
- 选择 `打开样式设置` 可调整粗细、透明度、颜色和 X 轴长度。
- 如果需要停止开发进程：

```powershell
npm run desktop:stop
```
